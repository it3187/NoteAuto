# NoteAuto - note自動下書き保存コンソール

このツールは、ローカルで記述したMarkdown形式の記事下書きを、noteの編集エディタ（`https://editor.note.com/new`）に自動で貼り付けて下書き保存するWebアプリケーションおよびCLI自動化パイプラインです。
美しいWebコンソール画面（UI）からの実行に加え、Gemini APIを用いた「AI-Slop（AI臭さ）自動スキャン ➔ 記事校正リライト ➔ 自動投稿」の一連のパイプライン処理に対応しています。

---

## ✨ 主な機能

1. **カバー画像（アイキャッチ）自動設定**
   - note.comの「みんなのフォトギャラリー」から、記事のタイトルに合う画像を自動的に1枚選択します。
   - noteの仕様変更に伴うトリミング（クロップ）画面の「保存」ボタン押下処理にも完全に対応しています。

2. **本文中への画像自動挿入**
   - 本文（Markdown）の中の任意の場所に、画像プレースホルダー `[IMAGE_PLACEHOLDER:ローカル画像パス]` を記述しておくだけで、Playwrightが自動的に段落の追加メニュー（＋メニュー）を起動し、指定された本物の画像ファイルをインラインでアップロード・挿入します。

3. **AI-Slop（AI臭さ）の検知・自動リライト（Python CLIパイプライン）**
   - `pipeline.py` を経由することで、記事のテーマ選定 ➔ 初回生成 ➔ AI臭さのスキャン判定 ➔ ルールに沿った自然な日本語への校正リライト ➔ note.comへの下書き保存までを、Gemini APIの無料枠制限（12秒間のスリープ）を考慮しながら自動で実行します。

---

## 📦 セットアップ方法

1. **必要なライブラリのインストール**:
   以下のコマンドを実行して、フロントエンドおよびバックエンドのNode.jsパッケージをインストールします。
   ```bash
   npm install
   ```

2. **Playwrightのブラウザ初期化**:
   Playwright用のChromiumブラウザ本体をインストールします。
   ```bash
   npx playwright install chromium
   ```

3. **Python依存関係のインストール (CLIパイプライン用)**:
   ```bash
   pip install -r requirements.txt
   ```

4. **環境変数の設定**:
   ルートディレクトリの `.env` ファイルに、使用するGemini APIのキーを設定します。
   ```env
   GEMINI_API_KEY=YOUR_API_KEY
   GEMINI_MODEL=gemini-2.5-flash
   ```

## 🚀 使い方

### A. Webコンソールから起動する場合
以下のコマンドを実行すると、ローカル開発サーバー（Vite + Express API）が立ち上がります。

```bash
npm run dev
```

1. 起動後、ブラウザで **`http://localhost:5173`** にアクセスします。
2. 画面上の入力フォームに、アップロードしたいMarkdownファイルの絶対パスを入力します。
3. **「下書きの自動作成を実行」** ボタンをクリックします。

### B. Python CLIパイプラインから起動する場合 (完全自動生成・校正・投稿)
Expressサーバーが起動している状態で、以下のコマンドを実行します。

```bash
python pipeline.py --theme "自動化が変えるブログ執筆の未来" --keywords "AI, 自動化, ブログ" --wordcount 1000 --upload
```

- `--theme`: 記事のテーマ（必須）
- `--upload`: 記事作成・校正完了後に自動で note.com へ下書き保存まで行うフラグ

## 🔐 初回起動時の注意（ログインセッションの保存）
- 初回起動時、noteにログインしていない場合は、自動的に開いたChromeブラウザ上にログイン画面が表示されます。
- ブラウザ上で手動でメールログイン ＆ セキュリティ認証（reCAPTCHA）を完了させてください。
- ログインが完了してエディタ画面が表示されると、**スクリプトが自動的にログイン完了を検知して、タイトルと本文の流し込みを再開します。**
- ログイン状態（Cookieやセッション）はローカルの `data/chrome_profile` フォルダ内に保存されるため、**2回目以降は認証なしで完全に自動で下書き保存が実行されます。**
